# p2p-leyu-journal

## 项目简介

本仓库用于归档和发布多个独立 HTML 页面。所有页面均为静态内容，不依赖于特定域名或外部服务。仓库旨在提供一个简洁、透明的页面存档与展示空间，方便查阅与维护。

## 目录结构

```
├── index.html          # 主入口页面（可选）
├── pages/              # 存放独立 HTML 页面的目录
│   ├── example1.html
│   ├── example2.html
│   └── ...
├── assets/             # 页面资源文件（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `pages/`：所有归档的独立 HTML 页面存放于此，每个页面具有独立的功能或内容。
- `assets/`：公共资源文件，供页面引用。

## 页面归档说明

- 每个 HTML 页面均为完整、自包含的静态文件，可直接通过浏览器打开或部署至静态托管服务。
- 页面之间无强制依赖关系，可单独访问。
- 归档内容不针对任何特定域名或网站，仅为通用性资料展示。

## 维护说明

- 新增页面：在 `pages/` 目录下创建新的 HTML 文件，并确保引用资源路径正确。
- 更新页面：直接修改对应 HTML 文件内容，注意保持版本一致性。
- 资源管理：公共资源建议放置于 `assets/` 下，避免冗余。
- 本仓库不涉及动态内容或后端逻辑，所有修改均可通过 Git 进行版本追踪。
